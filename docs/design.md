# Design Notes

This document summarizes Flow's product and UI direction.

## Product positioning

Flow is a privacy-conscious mobile health tracking app for logging body-flow patterns, hydration, reminders, and wellness reports.

The product should feel:

- calm
- practical
- private
- lightweight
- supportive

## Brand direction

**Product name:** Flow

The name emphasizes natural rhythm, movement, and routine. The app should be described as a health tracking tool, not as a joke app.

Recommended positioning:

> A privacy-conscious mobile health tracking app for logging body-flow patterns, hydration, reminders, and wellness reports.

## Visual direction

Core visual principles:

- soft health-oriented palette
- rounded cards
- large readable text
- calm spacing
- low-friction daily input
- minimal visual noise
- mobile-first layout

## UX priorities

Flow should help users:

1. record daily body-flow patterns quickly,
2. review weekly and monthly trends,
3. track hydration,
4. export personal records,
5. manage reminders and quiet hours.

## UI lessons from iteration

Earlier UI versions had issues such as:

- heavy gradients
- inconsistent spacing and shadows
- mixed icon styles
- unclear visual hierarchy
- partial bilingual text inconsistencies

The current showcase direction favors a softer and cleaner interface with consistent cards, soft green accents, and clearer information grouping.

## Accessibility notes

Areas to keep improving:

- text contrast
- touch target size
- reduced motion support
- clearer focus states
- consistent bilingual copy

## Future design improvements

- stronger onboarding flow
- improved chart readability
- cleaner report cards
- more consistent icon system
- better dark mode polish
- refined bilingual text
