# La Le Mo 完整测试流程

## ⚠️ 重要：Expo Go 测试不需要 EAS 登录！

如果你只是想用 Expo Go 测试应用，**完全不需要登录 EAS**。EAS 登录只在构建 APK 安装包时才需要。

---

## 方法一：使用 Expo Go 测试（推荐，最简单）⭐

### 步骤 1: 启动开发服务器
```bash
npm run start:go
```
或者
```bash
npm start -- --go
```

**不需要登录任何账户！**

### 步骤 2: 等待服务器启动
你会看到：
- ✅ Metro bundler 启动
- ✅ 显示二维码
- ✅ 显示 "Using Expo Go"
- ✅ 显示连接地址：`exp://10.0.0.222:8081` 或类似

### 步骤 3: 在手机上打开 Expo Go
1. 打开 Expo Go 应用
2. 扫描终端显示的二维码
3. 应用会自动加载

### 步骤 4: 如果扫描后显示错误页面
- 点击页面上的 **"Expo Go"** 按钮（白色按钮）
- 应用会在 Expo Go 中打开

---

## 方法二：构建 APK 安装包（需要 EAS 登录）

**只有当你想要构建 APK 文件时，才需要登录 EAS。**

### 步骤 1: 登录 EAS
```bash
npx expo login
```

### 步骤 2: 如果登录失败

#### 问题 1: 密码包含特殊字符
如果密码是 `www.lb520`，点号 `.` 可能导致问题。

**解决方案：**
1. 尝试用引号包裹密码（某些终端需要）
2. 或者使用交互式登录：
   ```bash
   npx expo login
   ```
   然后按提示输入用户名和密码

#### 问题 2: 账户不存在或密码错误
**解决方案：**
1. 访问 https://expo.dev 确认账户存在
2. 尝试重置密码
3. 或者创建新账户：
   ```bash
   npx expo register
   ```

#### 问题 3: 使用其他登录方式
```bash
# 使用 GitHub 登录
npx expo login --github

# 使用 Google 登录
npx expo login --google
```

### 步骤 3: 构建 APK
```bash
eas build --platform android --profile preview
```

## 常见问题

### Q: 为什么提示需要登录？
**A:** 你可能运行了需要 EAS 的命令（如 `eas build`）。如果只是测试，用 `npm start -- --go` 即可。

### Q: 登录时密码输入后没反应？
**A:** 这是正常的，密码输入时不会显示字符（安全考虑）。直接按回车即可。

### Q: 登录一直失败？
**A:** 
1. 确认账户在 https://expo.dev 可以登录
2. 尝试重置密码
3. 或使用 GitHub/Google 登录
